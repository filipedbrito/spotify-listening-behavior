# spotify-listening-behavior

This project explores **10 years of personal Spotify listening history** to understand how my music consumption behavior evolved over time.

The analysis is based on the **Top 100 most played tracks per year**, extracted from Spotify playlists using the **Spotify Web API** and analyzed with Python.

The goal is to explore how my listening behavior evolved across the years and answer questions such as:

- How much does my music taste change year to year?
- How many new artists appear in my yearly top tracks?
- Do my favorite songs persist over time?
- How do I balance exploration vs repetition in music?

This repository includes a sample of my personal listening data for demonstration purposes.

# Important Note (Playlist Ownership)

Spotify's official **"Your Top Songs YYYY"** playlists are owned by Spotify itself.

Because of this, they **may not appear when listing playlists through the Spotify API**.

To solve this limitation, the playlists were **manually duplicated and renamed** following the pattern:
```
top_tracks_2016
top_tracks_2017
...
top_tracks_2025
```

This ensures the playlists are **owned by the account and accessible via the API**.

# Spotify API Setup

To access Spotify data you must create a **Spotify Developer App**.

---

## 1 вЂ” Create a Spotify App

Go to:

https://developer.spotify.com/dashboard

Click **Create App** and fill something like:

```
App name: spotify-listening-behavior (suggestion)
App description: Personal Spotify listening analysis (suggestion)
Redirect URI: http://127.0.0.1:8888/callback
```

After creating the app you will receive:

- Client ID
- Client Secret

---

## 2 вЂ” Create a `.env` file

Create a `.env` file in the root of the project.

Example:
```
SPOTIFY_CLIENT_ID=your_client_id
SPOTIFY_CLIENT_SECRET=your_client_secret
SPOTIFY_REDIRECT_URI=http://127.0.0.1:8888/callback
```

These credentials are used by the scripts to authenticate requests to the Spotify API.

# Preparing Your Playlists

To reproduce the analysis:

1. Locate your **Spotify "Top Songs of YEAR" playlists**
2. Duplicate each playlist
3. Rename them using the pattern:
```
top_tracks_2016
top_tracks_2017
top_tracks_2018
```

This step ensures the playlists are **accessible through the Spotify API**.
